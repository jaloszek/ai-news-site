### Anthropic

- 🔥 [Claude Managed Agents — hosted long-horizon agent infrastructure (public beta)](https://www.anthropic.com/engineering/managed-agents) — Anthropic launched the public beta of Claude Managed Agents, a hosted service that runs long-running agents on your behalf. It decouples the "brain" (Claude + harness loop) from the "hands" (sandbox/execution environment), cutting time-to-first-token by ~60% at p50 and 90%+ at p95. Built-in: checkpointing, scoped permissions, VPC connectivity, credential vaulting that never exposes secrets to Claude-generated code, and persistent sessions across context windows.
  - Positioned as direct competition to n8n and Zapier — developers define agent tasks, tools, and guardrails while Anthropic handles orchestration, context, and error recovery. Enterprises can connect Claude to resources in their own VPCs without network peering. Early adopters include Notion, Rakuten, Asana, Vibecode, and Sentry.
  - Also: [The New Stack](https://thenewstack.io/with-claude-managed-agents-anthropic-wants-to-run-your-ai-agents-for-you/), [Testing Catalog](https://www.testingcatalog.com/anthropic-launches-claude-managed-agents-for-businesses/), [The Register](https://www.theregister.com/2026/04/09/anthropic_offers_to_host_ai/)

### Claude Code

- 🛠 [Claude Code v2.1.97 — Focus view toggle, status line refresh, MCP buffer fix](https://github.com/anthropics/claude-code/releases/tag/v2.1.97) — New `Ctrl+O` focus view toggle in `NO_FLICKER` mode shows a clean prompt + tool summary overlay. Status line gains a `refreshInterval` setting for periodic command re-runs. Fixes `--dangerously-skip-permissions` being incorrectly downgraded after approving protected path writes, and resolves an MCP buffer accumulation leak (~50 MB/hr) during server reconnections.
  - Also fixes resume picker search wiping, diffs disappearing on large files, and missing input from attachment messages during processing.

- 🛠 [Claude Code v2.1.98 — Vertex AI wizard, Perforce mode, Monitor tool, subprocess sandboxing + security fixes](https://github.com/anthropics/claude-code/releases/tag/v2.1.98) — Big quality-of-life and security release shipped late Thursday. Key additions:
  - **Vertex AI setup wizard**: Interactive GCP auth, project/region config, credential verification, and model pinning — accessible directly from the login screen. Parallels the Bedrock wizard from v2.1.92.
  - **Perforce mode** (`CLAUDE_CODE_PERFORCE_MODE`): Edit/Write/NotebookEdit commands now fail gracefully on read-only files with a `p4 edit` hint instead of silently overwriting — critical for teams on Perforce (common in game studios and large enterprises).
  - **Monitor tool**: Streams events from background scripts, enabling reactive workflows without polling.
  - **Subprocess sandboxing**: Linux PID namespace isolation when `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB` is set, plus `CLAUDE_CODE_SCRIPT_CAPS` to cap per-session script invocations.
  - **Security**: Fixed three Bash permission bypass vectors — backslash-escaped flags, compound commands, and env-var-prefixed read-only commands could all previously skip forced prompts.
  - Also: LSP client identification via `clientInfo`, OTEL `TRACEPARENT` propagation to Bash subprocesses, `/agents` tabbed interface, `/reload-plugins` without restart.
