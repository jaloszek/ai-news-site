## Tuesday, April 7

### Claude Code

- 🔥 [Claude Mythos Preview announced via Project Glasswing](https://red.anthropic.com/2026/mythos-preview/) — Anthropic quietly announced a research preview of their most capable model yet, described as "strikingly capable at computer security tasks." Rather than a public launch, access is invitation-only through "Project Glasswing," limited to 11 critical-infrastructure partners including Amazon, Google, Apple, Microsoft, Nvidia, and Cisco. The model autonomously finds zero-day vulnerabilities, writes working exploits, and chains multi-step privilege escalations — capabilities Anthropic judged too dangerous for general release.
  - Found thousands of zero-days including a 27-year-old OpenBSD TCP bug and a 16-year-old FFmpeg vuln missed by years of fuzzing; goal is to use it to patch defenses before attackers replicate the capability
  - Also: [Project Glasswing overview](https://www.anthropic.com/glasswing), [CrowdStrike founding-member post](https://www.crowdstrike.com/en-us/blog/crowdstrike-founding-member-anthropic-mythos-frontier-model-to-secure-ai/)

- 💜 [Messages API on Amazon Bedrock (research preview)](https://platform.claude.com/docs/en/release-notes/overview) — The first-party Claude Messages API endpoint (`/anthropic/v1/messages`) is now available on AWS infrastructure in `us-east-1`, using the identical request shape as the direct Claude API. This means code written for `api.anthropic.com` can be pointed at Bedrock with minimal changes, while running on AWS-managed infrastructure with zero operator data access.
  - Contact your Anthropic account executive to request access; standard Bedrock billing applies

## Wednesday, April 8

### Claude Code

- 🔥 [Claude Managed Agents in public beta](https://platform.claude.com/docs/en/managed-agents/overview) — Anthropic launched a fully managed agent harness for running Claude as an autonomous agent via API. It provides secure sandboxing, built-in tools (web search, code execution, file access), server-sent event streaming, and session management — effectively removing the need to wire up your own agent loop infrastructure. Use the `managed-agents-2026-04-01` beta header to access all endpoints.
  - Pairs naturally with the new `ant` CLI (below) for quick prototyping; enterprise teams can use it to deploy governed, sandboxed Claude agents without building infra from scratch
  - Also: [Overview docs](https://platform.claude.com/docs/en/managed-agents/overview)

- 🛠 [`ant` CLI for the Claude API](https://platform.claude.com/docs/en/api/sdks/cli) — A new official command-line client for the Anthropic API that enables faster scripted interaction, native integration with Claude Code sessions, and versioning of API resources (prompts, tools, agents) in YAML files checked into version control. Think of it as `gh` for the Claude API.
  - Complements Claude Code for teams that want to manage API resources as code, deploy prompts via CI/CD, or script multi-step agent workflows from the terminal

- 🛠 [CC v2.1.94 + v2.1.96: Amazon Bedrock support + hotfix](https://github.com/anthropics/claude-code/releases) — v2.1.94 added Amazon Bedrock via Mantle as a backend option and raised the default effort level to "high" for API-key and enterprise users (previously lower), alongside Slack integration improvements. v2.1.96 was a same-day hotfix resolving a `403 Authorization header is missing` error that broke Bedrock requests.
  - If you're on a corporate Bedrock deployment and hit auth errors, upgrading past 2.1.96 is required

## Thursday, April 9

### Claude Code

- 🔥 [Advisor tool in public beta](https://platform.claude.com/docs/en/agents-and-tools/tool-use/advisor-tool) — A new API feature that pairs a fast "executor" model for bulk token generation with a smarter "advisor" model that provides strategic guidance at key decision points mid-generation. Long-horizon agentic workloads get close to advisor-level quality at executor-model speeds and cost. Requires beta header `advisor-tool-2026-03-01`.
  - Useful for agentic pipelines where you want Opus 4.6 reasoning at critical junctures but Sonnet 4.6 throughput everywhere else — significant cost reduction for complex agents

- 🛠 [CC v2.1.97 + v2.1.98: Focus view toggle, Vertex AI wizard, subprocess sandboxing](https://github.com/anthropics/claude-code/releases) — v2.1.97 added a focus view toggle (`Ctrl+O` in no-flicker mode) showing just the prompt and final response without tool-call noise. v2.1.98 introduced an interactive Google Vertex AI setup wizard, subprocess sandboxing with PID namespace isolation on Linux, and a new Monitor tool for streaming events from background processes. Also fixed Bash tool permission vulnerabilities in both versions.
  - Subprocess sandboxing is a security improvement for users running Claude Code with `--dangerously-skip-permissions` in CI; Monitor tool enables watching long background tasks without polling

## Friday, April 10

### Claude Code

- 🛠 [CC v2.1.100 + v2.1.101: /team-onboarding command, enterprise TLS fixes](https://github.com/anthropics/claude-code/releases) — v2.1.101 added `/team-onboarding`, a new command that generates a personalized ramp-up guide for new teammates based on your local Claude Code usage patterns. ## Saturday, April 11

### Claude Code

- ⚠️ [1M token context window beta retiring April 30 for Claude Sonnet 4.5 and Sonnet 4](https://platform.claude.com/docs/en/release-notes/overview) — If you're using the `context-1m-2025-08-07` beta header with Sonnet 4.5 or original Sonnet 4, you have until April 30 to migrate. After that date, requests exceeding the standard 200k-token limit will return an error. Claude Sonnet 4.6 and Opus 4.6 already support 1M context natively at standard pricing with no beta header.
  - Action required: audit any agent pipelines or RAG apps using those models and either migrate to Sonnet 4.6/Opus 4.6 or implement chunking for inputs over 200k tokens
