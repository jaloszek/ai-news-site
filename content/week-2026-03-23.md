# AI News: Week of March 23, 2026

## Monday, March 23

### Claude Code

- 🛠 [Computer Use Comes to Claude Code](https://www.anthropic.com/news) — Claude Code (Pro and Max plans) can now open files, run dev tools, point, click, and navigate your screen with no setup required. Combined with Remote Control, you can send instructions from the Claude mobile app and Claude Code executes desktop operations on your Mac.
  - Claude Sonnet 4.6 scored 72.5% on OSWorld (up from <15% in late 2024), approaching human-level performance on tasks like spreadsheet navigation and web form completion

- 💜 [Claude Now Connects to Your World — Integrations Launch](https://anthropic.com/news/integrations) — Claude can now connect to external services and apps via a new integrations framework, letting it pull live data and take actions in connected tools without custom API plumbing.
  - Developers can expose their own services as Claude integrations via the API, enabling product-embedded AI that has real context about the user's environment

- 💜 [Claude and Productivity Platforms](https://anthropic.com/news/productivity-platforms) — Claude is now available natively inside major productivity platforms (details in linked post), enabling in-context AI assistance without switching tools.
  - Reduces context-switching cost for teams that already live inside productivity software

## Tuesday, March 24

### Claude Code

- 🛠 [Claude Code Auto Mode](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — New middle ground between manual approval and `--dangerously-skip-permissions`: safe actions run automatically while risky ones are blocked and Claude is redirected toward safer alternatives. Anthropic reports 93% of permission prompts were already being approved.
  - Auto Mode is the practical default for most solo developers — significantly fewer interruptions without the security surface of fully skipping permissions

- 🛠 [Scheduled Tasks — Background Cloud Jobs](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Claude Code can now run recurring tasks on Anthropic-managed infrastructure on a schedule. Give it a prompt, attached repos, a cron schedule, and environment connectors — it runs even when your computer is off.
  - Ideal for nightly PR reviews, dependency update PRs, CI failure triage, and other async developer chores

## Wednesday, March 25

### Claude Code

- 🛠 [Claude Can Now Create and Edit Files](https://anthropic.com/news/create-files) — Claude can create, edit, and manage files directly in conversations, persisting output beyond the chat window. Works across code files, documents, and structured data.
  - Removes the copy-paste step when iterating on code or documents — Claude writes directly to where the file needs to live

- 🛠 [`--bare` Flag for Scripted Calls](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — New `--bare` flag for `-p` (non-interactive) invocations skips hooks, LSP, plugin sync, and skill directory walks, making Claude Code faster and more predictable in CI and scripted pipelines.
  - Combine with `ANTHROPIC_API_KEY` or `apiKeyHelper` via `--settings` for fully headless operation in build systems

- 🛠 [`--channels` Permission Relay](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Channel servers declaring the permission capability can now forward tool approval prompts to your phone, so you can approve/deny Claude's actions remotely while a long task runs.
  - Pairs with computer use and scheduled tasks — approve a risky shell command from your phone while Claude is mid-task on your desktop

## Friday, March 27

### Claude Code

- 🛠 [Claude Code v2.1.86](https://github.com/anthropics/claude-code/releases) — Ships `X-Claude-Code-Session-Id` header on API requests (enables proxy-side session aggregation for billing/analytics), `.jj` and `.sl` VCS directory exclusion (Jujutsu and Sapling support), and improved scroll performance in large transcripts via a pure TypeScript layout engine.
  - The session ID header is a low-key but important addition for teams routing Claude Code through a corporate API gateway

- 🛠 [PowerShell Tool Preview (Windows)](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Opt-in preview of a PowerShell tool alongside new `ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS` env vars for third-party provider capability overrides (Bedrock, Vertex, Foundry).
  - Windows developers can now use Claude Code's shell integration without WSL for PowerShell-native workflows
