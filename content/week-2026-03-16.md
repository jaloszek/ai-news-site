# AI News: Week of March 16, 2026

## Monday, March 16

### Claude Code

- 🛠 [Claude Code v2.1.68–2.1.75 Releases](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Multiple releases shipped this week including transcript search (press `/` in transcript mode, `n`/`N` to step through matches), `StopFailure` hook event, and system-prompt caching improvements with ToolSearch and MCP tools.
  - Transcript search is especially useful for navigating large multi-turn sessions to find earlier tool outputs or decisions

## Tuesday, March 17

### Claude Code

- 🛠 [Subprocess Credential Scrubbing](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — New `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` env var strips credentials from subprocess environments, reducing accidental secret leakage when Claude Code shells out to build tools or scripts.
  - Recommended for CI/CD environments where Claude Code runs in shared infrastructure

- 🛠 [Session Display Names & Sparse Worktree Paths](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Sessions can now have human-readable display names; sparse worktree paths added for large monorepo workflows where checking out a full tree is impractical.
  - Sparse worktree support means Claude Code can now work efficiently in repos with hundreds of thousands of files

## Wednesday, March 18

### Claude Code

- 🛠 [Custom Auto-Memory Directory & Memory Timestamps](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Auto-memory can now be stored in a custom directory (useful for team-shared memory configs), and individual memory files get timestamps so you can tell when Claude learned something.
  - Memory timestamps help audit and clean up stale project context that may no longer reflect reality

## Thursday, March 19

### Claude Code

- 🔥 [Web Search & Code Execution Now Generally Available](https://docs.anthropic.com/en/release-notes/overview) — The web search tool and programmatic tool calling no longer require a beta header in API requests. Code execution is also free when used alongside web search or web fetch.
  - Developers can now build production apps with web search + code execution without managing beta flags or extra billing

## Friday, March 20

### Claude Code

- 🔥 [Claude Haiku 3 Deprecated — Retiring April 19, 2026](https://docs.anthropic.com/en/release-notes/overview) — Haiku 3 is being sunset; Haiku 4.5 is the recommended replacement, offering better performance at a similar price point.
  - Audit any production integrations still pinning `claude-haiku-3` — the model will stop responding after April 19

- 🔥 [Data Residency Controls Added (`inference_geo` Parameter)](https://docs.anthropic.com/en/release-notes/overview) — New `inference_geo` API parameter lets you specify where Claude inference runs geographically, supporting GDPR and data sovereignty requirements for EU-based customers.
  - Enterprise teams with data residency obligations can now enforce EU-only inference without routing through a separate API endpoint

## Saturday, March 21

### Claude Code

- 🛠 [Bug Fixes: OAuth Token Refresh & Voice Mode](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed multiple concurrent sessions requiring repeated re-authentication on OAuth token refresh, and fixed voice mode swallowing retry failures with a misleading "check your network" error.
  - The OAuth fix is significant for teams running parallel Claude Code sessions across multiple terminal windows

## Sunday, March 22

### Claude Code

- 🛠 [MCP Elicitation Support](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — MCP servers can now request additional information from the user mid-execution using the elicitation protocol, enabling interactive multi-step MCP workflows without pre-specifying all inputs.
  - Useful for MCP tools that need to ask clarifying questions (e.g., "which database table?") before executing a potentially destructive operation
