# AI News: Week of March 30, 2026

## Monday, March 30

### Claude Code

- 🛠 [v2.1.88 — Voice Mode Reliability Fixes](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed occasional input lag in voice mode, false "No speech detected" errors after releasing push-to-talk, and stale transcripts re-filling the prompt after submission. Also fixed `--continue` not resuming from the most recent point after `--compact`.
  - Voice mode is now reliable enough for continuous use in long coding sessions; the stale-transcript bug in particular was causing duplicate prompt submissions

- 🛠 [v2.1.88 — Startup ~500ms Faster](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — SessionStart hook execution is now deferred, cutting time-to-interactive by approximately 500ms. Memory usage in long-running sessions also improved: API stream buffers, agent context, and skill state are released after use.
  - Noticeable on machines where hooks do slow I/O (git status, network calls); also helps with RAM on large multi-agent sessions

- 🛠 [v2.1.88 — `PermissionDenied` Hook Added](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — New hook that fires after auto mode classifier denials. Return `{retry: true}` to let the model retry the denied action, enabling custom approval workflows without interrupting the agent.
  - Useful for building audit systems that can conditionally re-allow commands based on external policy checks

- 🛠 [v2.1.88 — Named Subagents in @ Typeahead](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Named subagents now appear in `@`-mention typeahead suggestions, making it easier to reference and hand off context to specific agents in multi-agent workflows.
  - Complements the existing subagent features; reduces context-switching friction in complex orchestration setups

- 🛠 [v2.1.88 — @-Mention Autocomplete Pre-warming](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — File suggestions for `@`-mentions now appear faster by pre-warming the index on startup and using session-based caching with background refresh.
  - Significant improvement on large monorepos where `@src/` used to pause before showing suggestions

- 🛠 [v2.1.88 — StructuredOutput Schema Cache Fix](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed a schema cache bug that caused approximately 50% failure rate in workflows using multiple `StructuredOutput` schemas in the same session.
  - Critical fix for any automation relying on structured JSON output across multiple tool calls in one session

- 🛠 [v2.1.88 — Nested CLAUDE.md Re-injection Fix](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed nested CLAUDE.md files being re-injected dozens of times in long sessions that read many files, causing context bloat and token waste.
  - Especially impactful for monorepos with per-directory CLAUDE.md instructions; long sessions were wasting significant context budget

- 🛠 [v2.1.88 — Hooks `if` Condition Fix for Compound Commands](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed `if` condition filtering not matching compound commands (`ls && git push`) or commands with env-var prefixes (`FOO=bar git push`).
  - Hook conditions were silently failing to fire on common shell patterns; this closes a gap in hook-based security policies

- 🛠 [v2.1.88 — `/stats` Fixes: History + Subagent Tokens](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Fixed `/stats` losing historical data beyond 30 days when the stats cache format changes, and fixed undercounting tokens by excluding subagent and fork usage.
  - Token reporting is now accurate for agentic workflows; historical data is preserved across CLI upgrades

- 🛠 [v2.1.88 — Thinking Summaries Off by Default](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Thinking summaries are no longer generated by default in interactive sessions. Set `showThinkingSummaries: true` in settings to restore the previous behavior.
  - Reduces noise in interactive sessions; useful to re-enable when debugging unexpected model reasoning

- 🛠 [v2.1.88 — Auto Mode Denial Visibility](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Auto mode denied commands now show a notification immediately and appear in the `/permissions` → Recent tab for review.
  - Makes it much easier to audit what the model tried to do and was blocked from; previously denials were silent

- 🛠 [v2.1.88 — Agent Task History Trimmed + Permission Classifier Fix](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Agent task message history is now trimmed after tasks complete, reducing memory growth in long agentic sessions. Also fixed a bash permission classifier bug where hallucinated match descriptions could incorrectly grant permissions.
  - The permission classifier fix is a security improvement: descriptions that don't correspond to real input rules are now rejected, closing a potential privilege escalation path

- 🛠 [v2.1.87 — Cowork Dispatch Fix](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Messages in Cowork Dispatch (the multi-session collaboration feature) were silently dropped; this patch ensures delivery.
  - If you use Cowork to route tasks across parallel Claude Code sessions, upgrade to 2.1.87 to avoid missed instructions

- 🛠 [v2.1.86 — Session ID Header & Observability](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Claude Code now sends an `X-Claude-Code-Session-Id` header on every API request, letting reverse proxies (LiteLLM, Helicone, custom gateways) group all calls from one session without body parsing.
  - Immediately useful for cost attribution and debugging multi-turn agentic runs behind a proxy

- 🛠 [v2.1.86 — JJ/Sapling VCS Support](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — `.jj` (Jujutsu) and `.sl` (Sapling) directories are now excluded from Grep and file autocomplete, preventing slowdowns for developers using these Git-compatible VCS tools.
  - Jujutsu in particular is gaining traction in the Rust/systems community; Claude Code now works cleanly in those repos

- 🛠 [v2.1.86 — Read Tool Token Savings](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — The Read tool now uses a compact line-number format and deduplicates unchanged re-reads, cutting token usage on file-heavy workflows.
  - Also: `@`-mentioned files no longer JSON-escape their content, reducing overhead on large file mentions

- 🛠 [v2.1.86 — Prompt Cache Improvements for Bedrock/Vertex/Foundry](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md) — Dynamic content removed from tool descriptions, improving cache hit rates for users running Claude Code through Bedrock, Vertex AI, or Azure Foundry.
  - Lower latency and reduced costs for enterprise deployments that route through cloud provider APIs
