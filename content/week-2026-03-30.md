# AI News: Week of March 30, 2026

## Monday, March 30

### Claude Code

- 🛠 [Claude Code v2.1.88](https://github.com/anthropics/claude-code/releases) — Major stability release with dozens of bug fixes and quality-of-life improvements.
  - Added `CLAUDE_CODE_NO_FLICKER=1` env var for flicker-free alt-screen rendering; named subagents now appear in `@` mention typeahead; fixed prompt cache misses in long sessions; fixed CRLF doubling on Windows; fixed `StructuredOutput` schema cache bug (~50% failure rate); fixed `/stats` undercounting tokens; thinking summaries no longer generated by default (set `showThinkingSummaries: true` to restore)

## Tuesday, March 31

### AI World

- 💜 [Australian Government and Anthropic Sign MOU for AI Safety and Research](https://www.anthropic.com/news) — Anthropic signed a Memorandum of Understanding with the Australian government covering AI safety cooperation and research collaboration.
  - Signals growing government-level engagement with Anthropic's safety-focused AI approach; follows similar agreements with other governments

## Wednesday, April 1

### Claude Code

- 🛠 [Claude Code v2.1.89](https://github.com/anthropics/claude-code/releases) — Headless/automation improvements and April Fools fun.
  - Added `"defer"` permission decision to `PreToolUse` hooks — headless sessions can now pause mid-run and resume later with `-p --resume`; `MCP_CONNECTION_NONBLOCKING=true` skips MCP connection wait in `-p` mode (bounded to 5s); `/buddy` Easter egg: a small animated creature that watches you code; fixed autocompact thrash loop; fixed `-p --resume` hangs on large deferred tool inputs; hook output over 50K chars now saved to disk instead of injected into context

- 🛠 [Claude Code v2.1.90](https://github.com/anthropics/claude-code/releases) — `/powerup` interactive lessons and critical bug fixes.
  - `/powerup` adds interactive animated lessons teaching Claude Code features; fixed infinite loop where rate-limit dialog would repeatedly auto-open and crash the session; fixed `--resume` causing full prompt-cache miss (regression since v2.1.69) for users with deferred tools, MCP servers, or custom agents; fixed `Edit`/`Write` failing when a PostToolUse format-on-save hook rewrites the file; hardened PowerShell permission checks (trailing `&` bypass, archive-extraction TOCTOU, and more)

## Thursday, April 2

### Claude Code

- 🛠 [Claude Code v2.1.91](https://github.com/anthropics/claude-code/releases) — MCP result size control, plugin executables, and transcript reliability fixes.
  - MCP tools can now annotate results with `_meta["anthropic/maxResultSizeChars"]` (up to 500K) to pass large payloads like DB schemas without truncation; plugins can ship executables under `bin/` and invoke them as bare Bash commands; added `disableSkillShellExecution` setting; multi-line prompts now work in `claude-cli://open?q=` deep links; fixed transcript chain breaks on `--resume` that could silently lose conversation history; Edit tool now uses shorter `old_string` anchors, reducing output tokens

## Friday, April 4

### Claude Code

- 🛠 [Claude Code v2.1.92](https://github.com/anthropics/claude-code/releases) — Remote settings enforcement, interactive Bedrock setup, per-model cost breakdown, and Linux sandboxing.
  - `forceRemoteSettingsRefresh` policy blocks startup until managed settings are freshly fetched (fail-closed for enterprise compliance); interactive Bedrock setup wizard guides AWS auth + region + model pinning from the login screen; `/cost` now shows per-model and cache-hit breakdown for subscription users; `/release-notes` is now an interactive version picker; Remote Control sessions use hostname as default name prefix; Write tool diff computation 60% faster on files with tabs/`&`/`$`; Linux ships `apply-seccomp` helper for sandboxing; `/tag` and `/vim` commands removed
